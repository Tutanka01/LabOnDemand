Administration

Utilisez la page Administration pour gerer les comptes, les templates, les runtimes et les labs actifs. Les templates actifs apparaissent dans le catalogue utilisateur.

Depannage

En cas d'erreur UI, actualisez la page puis verifiez l'onglet Audit et les logs API. Pour Kubernetes, controlez Stats cluster, Pods, Namespaces et Deployments.

Installation

Copiez le fichier .env.exemple vers .env, verifiez kubeconfig.yaml, puis lancez la pile avec docker compose up --build.

Cycle de vie des labs

Un lab peut etre en preparation, pret, en pause ou supprime. Ouvrir lance directement l'URL de l'application; Infos affiche les details Kubernetes.

Securite

Ne partagez pas de credentials, kubeconfigs ou fichiers .env. Les comptes locaux doivent utiliser un mot de passe fort.

Support

Indiquez le compte, l'heure, le lab, le namespace et le message d'erreur. Les logs d'audit permettent de retrouver les actions recentes.
